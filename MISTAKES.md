# MISTAKES.md - rp-mini

Mistakes to avoid and lessons learned. Add entries when an error happens twice or when the prompt says "oops".

1. (design phase) Don't gate codemaps by file size — large source files are where codemaps matter most. Gate only generated/minified content; cap codemap OUTPUT instead.
2. (design phase) Don't assume host capabilities from a reference implementation's config — CE's `multiAgentEnabled: false` was CE's choice, not a Codex platform limit. Verify against current platform docs.
3. (bead 1) `codex exec -m gpt-5.5-fast` fails on this ChatGPT-account setup ("model is not supported"); use the config default `gpt-5.5` with `-c model_reasoning_effort="medium"` for fast implementation runs. Also: codex exec can exit 0 despite a fatal model error — always check the log for `ERROR:` lines before trusting completion.
4. (bead 13) For this Codex CLI build, approval/sandbox flags such as `-a never` and `-s danger-full-access` must be placed before the `exec` subcommand when using `codex exec`; placing `--ask-for-approval` after `exec` exits with usage code 2. Do not combine `-a never` with `--dangerously-bypass-approvals-and-sandbox`; the bypass flag already owns approval/sandbox behavior.
5. (beads 11/14/16) `packages/codex-plugin/scripts/build-prompts.mjs` stamped a machine-absolute CLI path into the committed `config/mcp-servers.toml` every `pnpm build:prompts` run, while the committed file and test expect the `{{RP_MINI_SERVER_CLI}}` placeholder (install.sh renders the real path at install time). This bit three separate sessions as a mystery dirty file / failing test before being root-caused. Lessons: generators must reproduce committed content byte-for-byte (CI now runs `pnpm build:prompts && git diff --exit-code`); and when a test fails, fix the generator, not the assertion — the bead-16 engineer weakened the placeholder test to route around the gate, which would have failed on a clean CI checkout.
6. (beads 18-19 merges) `gh pr merge --squash --delete-branch` is a poor truth signal: it exits non-zero when the LOCAL branch cannot be deleted (e.g. held by a worktree) even though the remote merge succeeded, and piping it through `grep -v` masks the exit code entirely (one "merged" report was actually a branch-protection refusal: BEHIND after a same-day sibling merge). Verify merges with `gh pr view N --json state,mergedAt`, never with the merge command's exit code or stdout; use `gh pr update-branch N` when protection reports BEHIND.
7. (bead 22) `-c 'mcp_servers={}'` stopped disabling config-defined MCP servers in the current codex build — config.toml servers start anyway (superwall-mcp OAuth hung one run at startup; RepoPrompt MCP connected in another and the agent productively delegated to its context_builder). Treat the flag as unreliable: detect startup hangs by log-growth stall + near-zero CPU within ~2 minutes, kill and relaunch; consider a sandboxed CODEX_HOME (bead-13 pattern) when MCP must truly be off.
